Training the AI Model for Document Fuzzy Search

This guide provides detailed instructions for training the AI models used in the document fuzzy search system, focusing on the offline training process.
Overview of AI Components
The system uses two main AI components:

Vector Representation Model: Creates semantic embeddings of documents for similarity search
Document Classifier: Categorizes documents based on their content

Training Process
1. Vector Representation Model Training
The vector representation model uses TF-IDF (Term Frequency-Inverse Document Frequency) combined with dimensionality reduction:
Automatic Training During Indexing
The vector model is automatically trained when documents are indexed. The process includes:

Text Preprocessing:

Text extraction from documents
Cleaning and normalization
Tokenization
Stop word removal
Lemmatization or stemming


TF-IDF Vectorization:
python# This happens automatically when documents are indexed

vectorizer = TfidfVectorizer(
    max_df=0.9,          # Ignore terms that appear in >90% of documents
    min_df=2,            # Ignore terms that appear in <2 documents
    max_features=10000,  # Limit vocabulary size
    ngram_range=(1, 2)   # Include unigrams and bigrams
)
document_vectors = vectorizer.fit_transform(processed_texts)

Dimensionality Reduction:
python
# Reduce dimensions for faster searching
svd = TruncatedSVD(n_components=100)
reduced_vectors = svd.fit_transform(document_vectors)


Customizing the Vector Model
You can adjust the following parameters in the Settings tab:

Max Features: Number of terms in the vocabulary (default: 10000)
N-gram Range: Size of word sequences to consider (default: 1-2)
Number of Components: Dimensions after reduction (default: 100)

2. Document Classifier Training
The document classifier requires manual or rule-based labeling before training:
Step 1: Prepare Training Data

Manual Categorization:

Go to the Train tab
Select a document from the dropdown
Assign a category in the text field
Click "Assign Category"
Repeat for a representative sample of documents


Rule-based Categorization:

Go to the Train tab > Automatic Categorization
Select a rule type (Filename Contains, Extension Is, Content Contains)
Enter the rule value
Enter the category to assign
Click "Apply Rule"

Example rules:

Assign "Invoice" to documents with "invoice" in the filename
Assign "Contract" to documents with ".docx" extension
Assign "Technical" to documents containing "specification"



Step 2: Train the Classifier

Go to the Train tab
Review your categorized documents
Adjust the test size if needed (default: 20%)
Click "Train Classifier"
Review the training results (accuracy and classification report)
Save the model using the sidebar "Save Model" button

The classifier uses a Random Forest algorithm:
python
# Training process (simplified)

pipeline = Pipeline([
    ('vectorizer', TfidfVectorizer(max_features=5000, ngram_range=(1, 2))),
    ('classifier', RandomForestClassifier(n_estimators=100))
])

pipeline.fit(X_train, y_train)
Optimizing Model Performance
Vector Model Optimization

Adjust Preprocessing:

Try stemming instead of lemmatization for technical documents
Modify stop word lists for domain-specific content


TF-IDF Parameters:

Increase max_features for larger vocabularies
Adjust ngram_range to capture phrases (e.g., (1,3) for up to trigrams)
Modify min_df and max_df to filter rare or common terms


Dimensionality Reduction:

Increase components for more accurate but slower search
Decrease components for faster but less precise search



Classifier Optimization

Training Data Quality:

Aim for at least 15-20 examples per category
Ensure balanced distribution across categories
Use specific, non-overlapping categories


Model Parameters:

Adjust n_estimators in RandomForestClassifier
Modify max_depth to control tree complexity



Example Training Workflow
Here's a recommended workflow for training both models:

Index a representative document set:
# From the Index tab
1. Enter document directory path
2. Check "Include Subdirectories"
3. Click "Index Directory"

Create training data for classifier:
# From the Train tab

   - Rule: "Extension Is", Value: ".pdf", Category: "PDF Document"

   - Select important documents
   - Assign more specific categories

# From the Train tab
1. Verify category distribution
2. Set test size to 0.2 (20%)
3. Click "Train Classifier"
4. Review accuracy metrics

Optimize settings:
# From the Settings tab
1. Adjust similarity threshold based on search precision
2. Modify fuzzy match threshold based on query variations
3. Update vectorizer settings if needed

Save models:
# From the sidebar
1. Click "Save Index"
2. Click "Save Model"


Evaluating Model Performance
Vector Model Evaluation
The vector model's performance can be evaluated through search quality:

Precision: Are the top results relevant to queries?
Recall: Are all relevant documents being found?
Speed: How quickly do searches return results?

Adjust the similarity threshold in Settings to balance precision and recall.
Classifier Evaluation
The classifier provides performance metrics during training:

Accuracy: Overall percentage of correct classifications
Precision/Recall/F1: Performance metrics per category
Classification Report: Detailed breakdown of model performance

Target at least 80% accuracy for a useful classifier.
Retraining Models
Periodically retrain models when:

New document types are added to the system
Search performance degrades
Classification accuracy is insufficient
Significant new content is added to the index

For incremental updates:

Index new documents without rebuilding the entire index
Add classifications for new documents
Retrain the classifier with the expanded dataset

Handling Imbalanced Categories
When document categories have uneven distributions:

Sampling techniques:
pythonfrom imblearn.over_sampling import SMOTE
from imblearn.under_sampling import RandomUnderSampler

# Oversample minority classes
smote = SMOTE()
X_resampled, y_resampled = smote.fit_resample(X, y)

# Or undersample majority classes
under_sampler = RandomUnderSampler()
X_resampled, y_resampled = under_sampler.fit_resample(X, y)

Class weighting:
python# Adjust class weights inversely proportional to frequency
class_weights = {
    class_label: len(y) / (len(np.unique(y)) * np.sum(y == class_label))
    for class_label in np.unique(y)
}

# Apply weights to classifier
clf = RandomForestClassifier(class_weight=class_weights)
